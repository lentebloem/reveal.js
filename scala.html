Functional Programming

"Functional Programming is good because it leads to modules that can be combined freely."

Paraphrased from Why functional programming matters? -- John Hughes 1985
Functional Programming
Avoid side effects:
  • Modifies states are not predictable
  • Functions - return same output for the same parameters
  • Everything is an expression
  • Referential transparency: an expression can be replaced by its value without changing the behavior
Imperative Programming

Mutable variables and classes with internal states Example: POJO (Plain Old Java Object)

  • Variables can be changed by calling setter methods
  • Node reaching POJO instance can modify its variables
  • Prone to bugs

Some of the

Powerful statements in Scala

Operators are just methods


val b = 65

// + is a method in class Int, which can be called using
// the regular method call syntax...
val sum1 = a.+(b)

// ... or using the infix syntax
val sum2 = a + b

// You can use infix syntax with any method that takes one parameter
val result = List(2, 5, 13, 8) contains 4

Import statements can be anywhere


// Import everything from the package scala.concurrent
import scala.concurrent._

// Import scala.collection.mutable.Map as MutableMap
import scala.collection.mutable.{Map => MutableMap}

// Import everything from scala.concurrent, but hide JavaConversions
import scala.concurrent.{JavaConversions => _, _}

// Import math.sin and math.cos
// This is equivalent to Java's
// import static java.lang.Math.sin;
// import static java.lang.Math.cos;
import math.{sin, cos}

The body of the class is the constructor


class Hello(name: String) {
  // Statement executed as part of the constructor
  println("New instance with name: " + name)

  // Method which accesses the constructor argument
  def sayHello = println("Hello, " + name + "!")
}
This is equivalent to Java code: 

public class Hello {
    private final String name;

    public Hello(String name) {
        System.out.println("New instance with name: " + name);
        this.name = name;
    }

    public void sayHello() {
        System.out.println("Hello, " + name + "!");
    }
}

Objects instead of static methods


class Hello(name: String) {
  def sayHello = println("Hello, " + name + "!")
}

// Companion object of class Hello
object Hello {
  // Factory method
  def apply(name: String) = new Hello(name)
}
This is equivalent to Java code 

public class Hello {
    private final String name;

    public Hello(String name) {
        this.name = name;
    }

    public void sayHello() {
        System.out.println("Hello, " + name + "!");
    }

    public static Hello apply(String name) {
        return new Hello(name);
    }
}
